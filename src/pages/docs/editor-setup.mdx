## Поддержка синтаксиса

Tailwind CSS использует множество пользовательских CSS-правил, таких как `@tailwind`, `@apply` и `@screen`, и во многих редакторах это может вызывать предупреждения или ошибки, если эти правила не распознаются.

Решением этой проблемы почти всегда является установка плагина для вашего редактора/IDE для поддержки языка PostCSS вместо обычного CSS.

Если вы используете VS Code, наш официальный плагин [Tailwind CSS IntelliSense](https://marketplace.visualstudio.com/items?itemName=bradlc.vscode-tailwindcss) включает специальный языковой режим Tailwind CSS, который поддерживает все пользовательские at-правила и функции, которые использует Tailwind.

В некоторых случаях вам может потребоваться отключить встроенную обработку/проверку CSS, если ваш редактор очень строго придерживается синтаксиса, который он ожидает в ваших файлах CSS.

## IntelliSense для VS Code

Официальное расширение [Tailwind CSS IntelliSense](https://marketplace.visualstudio.com/items?itemName=bradlc.vscode-tailwindcss) для Visual Studio Code расширяет возможности разработки Tailwind, предоставляя пользователям расширенные функции, такие как автозаполнение, подсветка синтаксиса и линтинг.

<img src={require('@/img/intellisense.png').default.src} alt="Tailwind CSS IntelliSense extension for Visual Studio Code" width={1344} height={672} className="rounded-lg" />

- **Автозаполнение**. Интеллектуальные предложения по именам классов, а также [функции и директивы CSS](/docs/functions-and-directives).
- **Линтинг**. Подчеркивает ошибки и потенциальные ошибки как в вашем CSS, так и в разметке.
- **Предварительный просмотр при наведении**. Просмотрите полный CSS для имени класса Tailwind, наведя на него курсор.
- **Подсветка синтаксиса**. Предоставляет определения синтаксиса, чтобы функции Tailwind выделялись правильно.

Ознакомьтесь с проектом [на GitHub](https://github.com/tailwindcss/intellisense), чтобы узнать больше, или [добавьте его в Visual Studio Code](vscode:extension/bradlc.vscode-tailwindcss), чтобы начать работу.

## Автоматическая сортировка классов с Prettier

Мы поддерживаем официальный [плагин Prettier](https://github.com/tailwindlabs/prettier-plugin-tailwindcss) для Tailwind CSS, который автоматически сортирует ваши классы в соответствии с нашим [рекомендуемым порядком классов](/blog/automatic-class-sorting-with-prettier#how-classes-are-sorted).

<img src={require('@/img/prettier-banner.jpg').default.src} alt="" width={2400} height={819} className="rounded-lg" />

Он без проблем работает с пользовательскими конфигурациями Tailwind, а поскольку это всего лишь плагин Prettier, он работает везде, где работает Prettier, включая все популярные редакторы и IDE, и, конечно же, в командной строке.

```html
<!-- Before -->
<button class="text-white px-4 sm:px-8 py-2 sm:py-3 bg-sky-700 hover:bg-sky-800">...</button>

<!-- After -->
<button class="bg-sky-700 px-4 py-2 text-white hover:bg-sky-800 sm:px-8 sm:py-3">...</button>
```

Ознакомьтесь с плагином [на GitHub](https://github.com/tailwindlabs/prettier-plugin-tailwindcss), чтобы узнать больше и начать работу.

## JetBrains IDEs

IDE JetBrains, такие как WebStorm, PhpStorm и другие, включают поддержку интеллектуальных завершений Tailwind CSS в вашем HTML и при использовании `@apply`.

[Подробнее о поддержке Tailwind CSS в JetBrains IDEs &rarr;](https://www.jetbrains.com/help/webstorm/tailwind-css.html)
