## Обзор

Tailwind — это фреймворк для создания индивидуальных дизайнов, и для разных дизайнов требуются разные типографика, цвета, тени, брейкпоинты и многое другое.

Эти низкоуровневые решения по дизайну часто называют _дизайн-токенами_, и в проектах на Tailwind вы храните эти значения в _переменных темы_.

### Что такое переменные темы?

Переменные темы — это специальные CSS-переменные, определяемые с помощью директивы `@theme`, которые влияют на то, какие утилитарные классы доступны в вашем проекте.

Например, вы можете добавить новый цвет в проект, определив переменную темы, такую как `--color-mint-500`:

```css
/* [!code filename:app.css] */
@import "tailwindcss";

/* [!code highlight:4] */
@theme {
  --color-mint-500: oklch(0.72 0.11 178);
}
```

Теперь вы можете использовать утилитарные классы, такие как `bg-mint-500`, `text-mint-500` или `fill-mint-500` в вашем HTML:

```html
<!-- [!code filename:HTML] -->
<!-- [!code word:bg-mint-500] -->
<div class="bg-mint-500">
  <!-- ... -->
</div>
```

Tailwind также генерирует обычные CSS-переменные для ваших переменных темы, чтобы вы могли ссылаться на свои дизайн-токены в произвольных значениях или инлайн-стилях:

```html
<!-- [!code filename:HTML] -->
<!-- [!code word:var(--color-mint-500)] -->
<div style="background-color: var(--color-mint-500)">
  <!-- ... -->
</div>
```

Подробнее о том, как переменные темы сопоставляются с утилитарными классами, читайте в документации по [пространствам имён переменных темы](#theme-variable-namespaces).

#### Почему `@theme`, а не `:root`?

Переменные темы — это не _просто_ CSS-переменные — они также инструктируют Tailwind создавать новые утилитарные классы, которые вы можете использовать в HTML.

Поскольку они делают больше, чем обычные CSS-переменные, Tailwind использует специальный синтаксис, чтобы определение переменных темы всегда было явным. Переменные темы также должны определяться на верхнем уровне, а не внутри других селекторов или медиа-запросов, и специальный синтаксис позволяет это обеспечить.

Определение обычных CSS-переменных с помощью `:root` по-прежнему может быть полезно в проектах Tailwind, когда вы хотите определить переменную, не связанную с утилитарным классом. Используйте `@theme`, когда хотите, чтобы дизайн-токен напрямую сопоставлялся с утилитарным классом, и используйте `:root` для обычных CSS-переменных, которые не должны иметь соответствующих утилитарных классов.

### Связь с утилитарными классами

Некоторые утилитарные классы в Tailwind, такие как `flex` и `object-cover`, статичны и всегда одинаковы от проекта к проекту. Но многие другие зависят от переменных темы и существуют только потому, что вы их определили.

Например, переменные темы, определённые в пространстве имён `--font-*`, определяют все утилиты `font-family`, которые есть в проекте:

```css
/* [!code filename:./node_modules/tailwindcss/theme.css] */
@theme {
  /* [!code highlight:4] */
  /* prettier-ignore */
  --font-sans: ui-sans-serif, system-ui, sans-serif, "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol", "Noto Color Emoji";
  --font-serif: ui-serif, Georgia, Cambria, "Times New Roman", Times, serif;
  /* prettier-ignore */
  --font-mono: ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", "Courier New", monospace;
  /* ... */
}
```

Утилиты `font-sans`, `font-serif` и `font-mono` существуют по умолчанию только потому, что в стандартной теме Tailwind определены переменные `--font-sans`, `--font-serif` и `--font-mono`.

Если будет определена другая переменная темы, например `--font-poppins`, то появится и утилитарный класс `font-poppins`:

```css
/* [!code filename:app.css] */
@import "tailwindcss";

@theme {
  /* [!code highlight:2] */
  --font-poppins: Poppins, sans-serif;
}
```

```html
<!-- [!code filename:HTML] -->
<!-- [!code word:font-poppins] -->
<h1 class="font-poppins">Этот заголовок будет использовать Poppins.</h1>
```

Вы можете называть переменные темы как угодно в этих пространствах имён, и соответствующий утилитарный класс с таким же именем станет доступен в вашем HTML.

#### Связь с вариантами

Некоторые переменные темы используются для определения вариантов, а не утилит. Например, переменные темы в пространстве имён `--breakpoint-*` определяют, какие варианты брейкпоинтов существуют в вашем проекте:

```css
/* [!code filename:app.css] */
@import "tailwindcss";

@theme {
  /* [!code highlight:2] */
  --breakpoint-3xl: 120rem;
}
```

Теперь вы можете использовать вариант `3xl:*`, чтобы применять утилиту только при ширине вьюпорта 120rem и больше:

```html
<!-- [!code filename:HTML] -->
<!-- [!code word:3xl\:grid-cols-6] -->
<div class="3xl:grid-cols-6 grid grid-cols-2 md:grid-cols-4">
  <!-- ... -->
</div>
```

Подробнее о том, как переменные темы сопоставляются с утилитарными классами и вариантами, читайте в документации по [пространствам имён переменных темы](#theme-variable-namespaces).

### Переменные темы по умолчанию

Когда вы импортируете `tailwindcss` в начале вашего CSS-файла, он включает набор переменных темы по умолчанию, чтобы вы могли начать работу.

Вот что вы фактически импортируете, когда импортируете `tailwindcss`:

```css
/* [!code filename:node_modules/tailwindcss/index.css] */
@layer theme, base, components, utilities;

/* [!code highlight:2] */
@import "./theme.css" layer(theme);
@import "./preflight.css" layer(base);
@import "./utilities.css" layer(utilities);
```

Этот файл `theme.css` включает палитру цветов, шкалу типа, тени, шрифты и многое другое:

```css
/* [!code filename:node_modules/tailwindcss/theme.css] */
@theme {
  /* prettier-ignore */
  --font-sans: ui-sans-serif, system-ui, sans-serif, "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol", "Noto Color Emoji";
  --font-serif: ui-serif, Georgia, Cambria, "Times New Roman", Times, serif;
  --font-mono: ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", "Courier New", monospace;

  --color-red-50: oklch(0.971 0.013 17.38);
  --color-red-100: oklch(0.936 0.032 17.717);
  --color-red-200: oklch(0.885 0.062 18.334);
  /* ... */

  --shadow-2xs: 0 1px rgb(0 0 0 / 0.05);
  --shadow-xs: 0 1px 2px 0 rgb(0 0 0 / 0.05);
  --shadow-sm: 0 1px 3px 0 rgb(0 0 0 / 0.1), 0 1px 2px -1px rgb(0 0 0 / 0.1);
  /* ... */
}
```

Вот почему утилиты типа `bg-red-200`, `font-serif` и `shadow-sm` существуют из коробки — они управляются темой по умолчанию, а не жестко закодированы во фреймворке, как `flex-col` или `pointer-events-none`.

Для полного списка всех переменных темы по умолчанию см. [справочник переменных темы по умолчанию](#default-theme-variable-reference).

## Кастомизация вашей темы

Переменные темы по умолчанию очень универсальны и подходят для создания совершенно разных дизайнов, но они все еще являются лишь отправной точкой. Очень часто приходится настраивать такие вещи, как цветовая палитра, шрифты и тени, чтобы создать именно тот дизайн, который вы задумали.

### Расширение темы по умолчанию

Используйте `@theme` для определения новых переменных темы и расширения темы по умолчанию:

```css
/* [!code filename:app.css] */
@import "tailwindcss";

@theme {
  /* [!code highlight:2] */
  --font-script: Great Vibes, cursive;
}
```

Это делает доступным новый класс утилиты `font-script`, который вы можете использовать в вашем HTML, точно так же, как стандартные утилиты `font-sans` или `font-mono`:

```html
<!-- [!code filename:HTML] -->
<!-- [!code word:font-script] -->
<p class="font-script">This will use the Great Vibes font family.</p>
```

Узнайте больше о том, как переменные темы сопоставляются с различными классами утилит и вариантами в документации [пространства имен переменных темы](#theme-variable-namespaces).

### Переопределение темы по умолчанию

Переопределите значение переменной темы по умолчанию, переопределив её внутри `@theme`:

```css
/* [!code filename:app.css] */
@import "tailwindcss";

@theme {
  /* [!code highlight:2] */
  --breakpoint-sm: 30rem;
}
```

Теперь вариант `sm:*` будет срабатывать при ширине вьюпорта 30rem вместо стандартных 40rem:

```html
<!-- [!code filename:HTML] -->
<!-- [!code word:sm\:grid-cols-3] -->
<div class="grid grid-cols-1 sm:grid-cols-3">
  <!-- ... -->
</div>
```

Чтобы полностью переопределить всё пространство имён в теме по умолчанию, установите всё пространство имён в `initial` с помощью специального синтаксиса со звёздочкой:

```css
/* [!code filename:app.css] */
@import "tailwindcss";

@theme {
  /* [!code highlight:2] */
  --color-*: initial;
  --color-white: #fff;
  --color-purple: #3f3cbb;
  --color-midnight: #121063;
  --color-tahiti: #3ab7bf;
  --color-bermuda: #78dcca;
}
```

При этом все утилиты по умолчанию, использующие это пространство имён _(например, `bg-red-500`)_, будут удалены, и будут доступны только ваши пользовательские значения _(например, `bg-midnight`)_.

Узнайте больше о том, как переменные темы сопоставляются с различными классами утилит и вариантами в документации [пространства имен переменных темы](#theme-variable-namespaces).

### Использование пользовательской темы

Чтобы полностью отключить тему по умолчанию и использовать только пользовательские значения, установите глобальное пространство имён переменных темы в `initial`:

```css
/* [!code filename:app.css] */
@import "tailwindcss";

@theme {
  /* [!code highlight:2] */
  --*: initial;

  --spacing: 4px;

  --font-body: Inter, sans-serif;

  --color-lagoon: oklch(0.72 0.11 221.19);
  --color-coral: oklch(0.74 0.17 40.24);
  --color-driftwood: oklch(0.79 0.06 74.59);
  --color-tide: oklch(0.49 0.08 205.88);
  --color-dusk: oklch(0.82 0.15 72.09);
}
```

Теперь ни один из утилитарных классов по умолчанию, управляемых переменными темы, не будет доступен, и вы сможете использовать только утилитарные классы, соответствующие вашим пользовательским переменным темы, такие как `font-body` и `text-dusk`.

### Определение ключевых кадров анимации

Определите правила `@keyframes` для ваших переменных темы `--animate-*` внутри `@theme`, чтобы включить их в сгенерированный CSS:

```css
/* [!code filename:app.css] */
@import "tailwindcss";

@theme {
  --animate-fade-in-scale: fade-in-scale 0.3s ease-out;

  @keyframes fade-in-scale {
    0% {
      opacity: 0;
      transform: scale(0.95);
    }
    100% {
      opacity: 1;
      transform: scale(1);
    }
  }
}
```

Если вы хотите, чтобы ваши пользовательские правила `@keyframes` всегда включались, даже когда не добавляется переменная темы `--animate-*`, определите их вне `@theme`.

### Ссылки на другие переменные

При определении переменных темы, которые ссылаются на другие переменные, используйте опцию `inline`:

```css
/* [!code filename:app.css] */
@import "tailwindcss";

/* [!code word:inline] */
@theme inline {
  --font-sans: var(--font-inter);
}
```

При использовании опции `inline` утилитарный класс будет использовать _значение_ переменной темы вместо ссылки на саму переменную темы:

```css
/* [!code filename:dist.css] */
.font-sans {
  font-family: var(--font-inter);
}
```

Без использования `inline` ваши утилитарные классы могут разрешаться в неожиданные значения из-за того, как переменные разрешаются в CSS.

Например, этот текст будет использовать `sans-serif` вместо `Inter`, как вы могли бы ожидать:

```html
<!-- [!code filename:HTML] -->
<!-- [!code word:--font-sans\: var(--font-inter, sans-serif);] -->
<div id="parent" style="--font-sans: var(--font-inter, sans-serif);">
  <!-- [!code word:--font-inter\: Inter; font-family\: var(--font-sans);] -->
  <div id="child" style="--font-inter: Inter; font-family: var(--font-sans);">
    This text will use the sans-serif font, not Inter.
  </div>
</div>
```

Это происходит потому, что `var(--font-sans)` разрешается там, где определена `--font-sans` _(на `#parent`)_, а `--font-inter` там не имеет значения, поскольку она не определена до более глубокого уровня в дереве _(на `#child`)_.

### Генерация всех CSS-переменных

По умолчанию в финальном CSS-выводе генерируются только используемые CSS-переменные. Если вы хотите всегда генерировать все CSS-переменные, вы можете использовать опцию темы `static`:

```css
/* [!code filename:app.css] */
/* [!code word:static] */
@import "tailwindcss";

@theme static {
  --color-primary: var(--color-red-500);
  --color-secondary: var(--color-blue-500);
}
```

### Обмен между проектами

Поскольку переменные темы определяются в CSS, их обмен между проектами — это просто вопрос помещения их в отдельный CSS-файл, который вы можете импортировать в каждом проекте:

```css
/* [!code filename:./packages/brand/theme.css] */
@theme {
  --*: initial;

  --spacing: 4px;

  --font-body: Inter, sans-serif;

  --color-lagoon: oklch(0.72 0.11 221.19);
  --color-coral: oklch(0.74 0.17 40.24);
  --color-driftwood: oklch(0.79 0.06 74.59);
  --color-tide: oklch(0.49 0.08 205.88);
  --color-dusk: oklch(0.82 0.15 72.09);
}
```

Затем вы можете использовать `@import` для включения переменных темы в другие проекты:

```css
/* [!code filename:./packages/admin/app.css] */
@import "tailwindcss";
/* [!code highlight:2] */
@import "../brand/theme.css";
```

Вы можете поместить общие переменные темы таким образом в отдельный пакет в настройках монорепозитория или даже опубликовать их в NPM и импортировать как любые другие сторонние CSS-файлы.

## Использование переменных темы

Все ваши переменные темы превращаются в обычные CSS-переменные при компиляции CSS:

```css
/* [!code filename:dist.css] */
:root {
  /* prettier-ignore */
  --font-sans: ui-sans-serif, system-ui, sans-serif, "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol", "Noto Color Emoji";
  --font-serif: ui-serif, Georgia, Cambria, "Times New Roman", Times, serif;
  --font-mono: ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", "Courier New", monospace;

  --color-red-50: oklch(0.971 0.013 17.38);
  --color-red-100: oklch(0.936 0.032 17.717);
  --color-red-200: oklch(0.885 0.062 18.334);
  /* ... */

  --shadow-2xs: 0 1px rgb(0 0 0 / 0.05);
  --shadow-xs: 0 1px 2px 0 rgb(0 0 0 / 0.05);
  --shadow-sm: 0 1px 3px 0 rgb(0 0 0 / 0.1), 0 1px 2px -1px rgb(0 0 0 / 0.1);
  /* ... */
}
```

Это позволяет легко ссылаться на все ваши дизайн-токены в любом из ваших пользовательских CSS или инлайн-стилей.

### С пользовательским CSS

Используйте переменные темы для доступа к вашим дизайн-токенам при написании пользовательского CSS, которому нужно использовать те же значения:

```css
/* [!code filename:app.css] */
/* [!code word:var(--text-base)] */
/* [!code word:var(--color-gray-700)] */
/* [!code word:var(--text-2xl)] */
/* [!code word:var(--font-weight-semibold)] */
/* [!code word:var(--color-gray-950)] */
/* [!code word:var(--text-xl)] */
@import "tailwindcss";

@layer components {
  .typography {
    p {
      font-size: var(--text-base);
      color: var(--color-gray-700);
    }

    h1 {
      font-size: var(--text-2xl--line-height);
      font-weight: var(--font-weight-semibold);
      color: var(--color-gray-950);
    }

    h2 {
      font-size: var(--text-xl);
      font-weight: var(--font-weight-semibold);
      color: var(--color-gray-950);
    }
  }
}
```

Это часто полезно при стилизации HTML, которым вы не управляете, например, контента Markdown, поступающего из базы данных или API и отображаемого в HTML.

### С произвольными значениями

Использование переменных темы в произвольных значениях может быть полезным, особенно в сочетании с функцией `calc()`.

```html
<!-- [!code filename:HTML] -->
<!-- [!code word:rounded-xl] -->
<div class="relative rounded-xl">
  <!-- [!code word:rounded-\[calc(var(--radius-xl)-1px)\]] -->
  <div class="absolute inset-px rounded-[calc(var(--radius-xl)-1px)]">
    <!-- ... -->
  </div>
  <!-- ... -->
</div>
```

В приведенном выше примере мы вычитаем 1px из значения `--radius-xl` на вложенном внутреннем элементе, чтобы убедиться, что у него концентрический радиус границы.

### Ссылки в JavaScript

В большинстве случаев, когда вам нужно ссылаться на переменные темы в JS, вы можете просто использовать CSS-переменные напрямую, как и любое другое CSS-значение.

Например, популярная библиотека [Motion](https://motion.dev/docs/react-quick-start) для React позволяет анимировать к CSS-переменным значениям и от них:

```jsx
// [!code filename:JSX]
// [!code word:var(--color-blue-500)]
<motion.div animate={{ backgroundColor: "var(--color-blue-500)" }} />
```

Если вам нужен доступ к разрешённому значению CSS-переменной в JS, вы можете использовать `getComputedStyle` для получения значения переменной темы в корне документа:

```js
// [!code filename:spaghetti.js]
let styles = getComputedStyle(document.documentElement);
let shadow = styles.getPropertyValue("--shadow-xl");
```
